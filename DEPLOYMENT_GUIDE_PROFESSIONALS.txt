# 🚀 Admin Professionals Manager - Deployment Guide

## Quick Summary
You now have a complete admin panel for managing professional account approvals and rejections. Professionals can be approved with automatic email confirmation, or rejected with a custom reason message.

## 📋 What Was Built

### Component: ProfessionalsManager
- Professional list with filtering (All, Pending, Approved, Rejected)
- Search functionality (name/email)
- Details modal showing all professional information
- Approve button → sends approval email + in-app notification
- Reject button → modal for reason → sends rejection email + notification
- Status badges with color coding
- Responsive, animated UI matching CareLink's design

### Edge Functions
1. **send-approval-email**: HTML email sent when professional approved
2. **send-rejection-email**: HTML email sent when professional rejected with reason

### Integration
- Integrated into AdminPanel.tsx under "Professionnels" tab
- No breaking changes to existing code

## 🔧 Deployment Steps

### Step 1: Configure Environment Variables
Add to your Supabase project settings:

```env
# Required for email sending
RESEND_API_KEY=sk_live_xxxxx (or sk_test for testing)
APP_URL=https://app.carelinks.app  # or your domain
```

### Step 2: Deploy Edge Functions
```bash
cd C:\carelink

# Deploy approval email function
supabase functions deploy send-approval-email

# Deploy rejection email function
supabase functions deploy send-rejection-email
```

### Step 3: Build Web App
```bash
pnpm build
# Output: dist/ folder ready for deployment
```

### Step 4: Deploy Web App
Deploy the `dist/` folder to your hosting (Vercel, Netlify, etc.)

## 🐛 Troubleshooting

### "Email not sending"
1. Check RESEND_API_KEY is set in Supabase project
2. Verify API key is valid (test at resend.com)
3. Check edge function logs: `supabase functions logs send-approval-email`
4. Verify professional has valid email address

### "Notification not appearing"
1. Check `notifications` table has new records
2. Verify professional can access the app
3. Check that `user_id` matches professional's Supabase auth ID

### "Modal not opening"
1. Check browser console for errors
2. Verify data is loading from Supabase
3. Clear browser cache

### "Build failing"
1. Run `pnpm install` to ensure dependencies installed
2. Delete `node_modules` and `.pnpm` folders
3. Run `pnpm install` again
4. Try `pnpm build` again

## 📊 Database Tables Used

### professionals
```sql
-- Key columns for this feature:
- id: UUID (primary key)
- verification_status: 'pending' | 'approved' | 'rejected'
- rejection_reason: text (nullable)
- verified_at: timestamp (nullable)
- specialty: 'nurse' | 'psychologist' | 'yoga_instructor' | 'physiotherapist'
- user_id: UUID (foreign key to profiles/auth.users)
```

### notifications
```sql
-- Created for each approval/rejection:
- id: UUID (primary key)
- user_id: UUID (professional's user ID)
- type: 'approval' | 'rejection'
- title: text
- message: text
- read: boolean (default: false)
- created_at: timestamp
```
