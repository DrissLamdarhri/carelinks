# Edge Functions, Realtime & Push

## Edge Functions (`supabase/functions/`, Deno)

### `server/index.tsx` — the KV marketplace backend (Path A)

Hono app, all routes prefixed `/make-server-aa5d1aa6/*`. Backed by service-role client + `kv_store.tsx`
(table `kv_store_aa5d1aa6`, `key TEXT PK, value JSONB`). This is the **web/demo** backend; the mobile app
does not use it.

Route groups:
- **Health:** `GET /health`.
- **Auth:** `POST /auth/patient-signup`, `POST /auth/pro-signup` — create the auth user (service role) + KV
  profile, **best-effort dual-write** to the relational `profiles`/`patients`/`professionals` tables.
- **Users:** `GET/PUT /users/:id`.
- **Professionals:** `GET /professionals`, `GET/PUT /professionals/:id`, `PUT /professionals/:id/online`.
- **Requests (bidding):** `POST/GET /requests`, `GET /requests/:id`, `PUT /requests/:id/cancel`. Creating a
  request also dual-writes a `bookings` row and broadcasts (below).
- **Offers (bids):** `POST/GET /requests/:id/offers`, `PUT /offers/:id/accept` (creates KV booking + dual-writes
  bid acceptance to tables), `PUT /offers/:id/reject`.
- **Bookings:** `GET /bookings`, `GET /bookings/:id`, `PUT /bookings/:id/complete`, `POST /bookings/:id/rate`.
- **Admin:** `GET /admin/stats` (reads relational tables), `/admin/professionals/pending`, approve/reject,
  `/admin/bookings/recent`, `/admin/notifications`, `POST /admin/log-booking`, `POST /admin/login`.

**Admin auth:** every admin route accepts header `X-Admin-Key: carelink-admin-2024` (hardcoded) **or** a real
admin profile via `requireAdmin()`. `/admin/login` returns `{ token: "carelink-admin-2024" }` for the demo
creds. ⚠️ See [tech-debt-and-security.md](tech-debt-and-security.md).

**Dual-write:** the function attempts best-effort writes into relational tables (`syncProfile`,
`syncPatient`, `syncProfessional`, `createSupabaseBooking`, `createSupabaseBid`, `acceptSupabaseBid`) so
Path-A activity partially shows up in Path B — but this is one-directional and lossy, not true sync.

### `server/kv_store.tsx`

Autogenerated KV interface over `kv_store_aa5d1aa6` using the service-role key: `get/set/del`,
`mget/mset/mdel`, `getByPrefix`.

### `send-approval-email/` & `send-rejection-email/`

Deno `serve` functions. `POST { email, name, specialty }` / `{ email, name, reason }` → French HTML email via
**Resend** (`RESEND_API_KEY`, from `admin@carelinks.app`, CTA to `${APP_URL}`). CORS `*`; they do **not**
authenticate the caller.

## Realtime

Two distinct realtime mechanisms coexist.

### Postgres CDC (`postgres_changes`) — Path B, the production one

Hooks live in `mobile-app/lib/db/realtime.ts`, `src/lib/db/realtime.ts`, `shared/db/realtime.ts` (three
near-identical copies; `shared` is the most complete, mobile adds demo-ID guarding).

| Hook | Channel | Table / filter |
|------|---------|----------------|
| `useBookingBids(bookingId)` | `bids:booking:{id}` | `bids` where `booking_id` — sorted by `price_mad` |
| `usePatientBookings(patientId)` | `bookings:patient:{id}` | `bookings` where `patient_id` |
| `useOpenBookingsBySpecialty(spec)` | `bookings:specialty:{spec}` | `bookings` where `specialty`, `status=open` |
| `useBookingMessages(bookingId)` | `messages:booking:{id}` | `messages` INSERT (dedupes by id) |
| `useUserNotifications(userId)` | `notifications:user:{id}` | `notifications` where `user_id`, computes unread |

Subscribe on `useFocusEffect`; tear down with `supabase.removeChannel`. Requires the tables to be in the
`supabase_realtime` publication (done in `triggers.sql` / enabled in the dashboard).

### Broadcast channels (client `channel.send` / server REST)

- **Live tracking:** channel `tracking:{bookingId}`, event `position` `{lat,lng,at}`. Pro side
  (`LiveTrackingChannel mode="broadcast"`) streams `expo-location` High-accuracy every 10s; patient side
  (`mode="watch"`) receives → moves the map's pro dot.
- **Subscriptions:** channel `private-user-{userId}`, events `subscription.updated` / `subscription.rollback`.
- **Admin notifications:** channel `admin-notifications`, event `booking_event`.
- **Edge Function (Path A) broadcasts** via REST helper (`broadcast()` → `/realtime/v1/api/broadcast` with
  service-role key): channel `carelink-requests`/event `new_request` (fan out to pros), and
  `request-{requestId}`/event `new_offer` (notify waiting patient). These are the **KV** equivalents of the
  Path-B bid channels — same InDrive flow, different transport/terminology.

## Push notifications

- **Mobile (`mobile-app/lib/push-native.ts`):** Expo push. `registerExpoPushToken(userId)` (disabled in Expo
  Go), requests permission, `getExpoPushTokenAsync()`, **upserts `push_subscriptions {user_id,
  expo_push_token, platform}`**. `configureNotifications()` sets the foreground handler.
- **Web (`src/lib/push.ts`):** Web Push via `/sw.js` + VAPID. ⚠️ `VAPID_PUBLIC_KEY` placeholder — not wired.
- **Schema (`supabase/push.sql`):** `push_subscriptions (user_id, endpoint, payload)` + RLS `push_self`.
  ⚠️ **Column mismatch:** SQL has `endpoint`/`payload` (web shape); mobile writes `expo_push_token`/`platform`.
  Reconcile before relying on mobile push.
- **Admin booking notifications** (`mobile-app/lib/admin/booking-notifications.ts`): `notifyAdminNewBooking`
  computes `alertLevel` then calls Edge `POST /admin/log-booking` (service-role write to `admin_booking_logs`
  + admin `notifications`); `broadcastAdminNotification` fans out on `admin-notifications`. Read/query APIs:
  `fetchAdminBookings`, `fetchPriorityBookings`, `fetchBookingStats`.
